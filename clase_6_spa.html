La Trampa del Accuracy

Hasta ahora, hemos evaluado los modelos respondiendo: "¿Qué porcentaje de aciertos tuvo?". Esto se conoce como Accuracy (Exactitud). Sin embargo, en la vida real, el Accuracy puede mentirnos severamente cuando los datos están desbalanceados.

🚨 El Problema Médico

Imagina una enfermedad muy rara que afecta a 1 de cada 100 pacientes. Entrenas un modelo de IA que es "vago" y simplemente predice "SANOS" para todos.

  • Pacientes Totales: 100
  • Sanos: 99 | Enfermos: 1
  • Aciertos del modelo: 99/100
  • ➔ Accuracy = 99% 🌟

¿Es un buen modelo?

¡NO! Es un modelo inútil. Falló miserablemente en su único propósito: encontrar al paciente enfermo (el que realmente importa).

Por eso, necesitamos evaluar dónde se equivoca el modelo, no solo cuánto acierta.

Curvas ROC y AUC

La curva ROC (Receiver Operating Characteristic) grafica la relación entre la Tasa de Verdaderos Positivos y la Tasa de Falsos Positivos a través de distintos umbrales de decisión.

El área bajo la curva se conoce como AUC.

  • AUC = 0.5: El modelo es como tirar una moneda al aire (línea diagonal).
  • AUC = 0.7 - 0.8: Modelo aceptable a bueno.
  • AUC = 1.0: Modelo perfecto (Pasa por la esquina superior izquierda).

Validación Cruzada (K-Fold) y Tuning

Separar los datos en "Entrenamiento" y "Prueba" una sola vez puede ser un golpe de suerte. Para asegurar que el modelo es robusto, usamos K-Fold Cross-Validation, iterando y rotando el bloque de prueba varias veces.

🔄 1. K-Fold CV

Calculamos la métrica final como el promedio de todas las iteraciones. Así garantizamos que el modelo generaliza bien.

🎛️ 2. Grid Search CV

Los modelos tienen configuraciones "manuales" llamadas Hiperparámetros (ej: la profundidad de un árbol). Grid Search es una fuerza bruta automatizada:

params = {
  'max_depth': [3, 5, 10],
  'criterion': ['gini', 'entropy']
}

Scikit-Learn probará las 6 combinaciones (3x2) usando Validación Cruzada y te devolverá el mejor modelo posible.
